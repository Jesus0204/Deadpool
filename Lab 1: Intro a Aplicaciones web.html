¿Cuál es la diferencia entre Internet y la World Wide Web?

De acuerdo a la BBC (2019), el Internet es una red de computadoras que estan conectadas entre sí. La web es una colección de páginas que se asienta sobre esa red de computadoras

¿Cuáles son las partes de un URL?

Una página web primero tiene el protocolo (por ejemplo http) o esquema. HTTPS incluye un cifrado

Despúes está el subdominio. Este determina qué contenido mostrar, el más común es www, pero existen otros como blog.

El dominio de segundo nivel se pone el nombre de tu empresa

El dominio de nivel superior indica que tipo de organización tienes (puede ser gov, edu, org, etc).

También existe la ruta, que dicta qué página específica abrir del sitio web.

Despúes puede estar el documento que se encuentra en esa ruta (en páginas estáticas)

Si se puede hacer una consulta, esta se representa con un signo de interrogación. También se le pueden poner parámetros.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

El método GET es una petición a un recurso específico al servidor. No se pueden enviar datos.

Con POST se pueden enviar datos al servidor por medio del cuerpo

Es parecido al POST, pero siempre realiza la misma acción. Un post publica encima, es como la inserción de un nuevo registro.

DELETE puedes eliminar un recurso específico.

Es similar al PUT, ya que actualiza la parte de un registro. Son modificaciones parciales

El HEAD como el GET pero no regresa ningún contenido. Solo se quiere saber el código HTTP de respuesta.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Hay dos formas de enviar un formulario. Con GET, los valores que se envian son visibles en URL, donde con POST se envian en encabezados HTTP y no so0n visibles. Todos los datos confidenciales se deben de enviar de esta forma.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Si es responsabilidad del desarrollador, ya que lo que pasó fue que se buscó la página pero como no se encontró, se devuelve ese error. Cuando se corriga y se suba la página al servidor, ya no va a aparecer el error. El usuario no tiene control sobre esto.

Referencia

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

En este caso es un poco más complicado, porque significa que algo fallo con la conexión al servidor (como su base de datos). Sin embargo, el desarrollador tiene que ver qué es lo que causo esa ruptura y corregir el error para que la página se despliege como debe ser.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

El desarrollo web va evolucionando, y es por esto que se esta migrando del HTML4 al HTML5. El HTML5 tiene ciertos atributos nuevos, pero esto no signfica que hayan quitado los de HTML4. Es por eso que estos atributos estan despreciados.

Unos ejemplos de estos atributos son:

  • big (ahora se hace con CSS)
  • center (ahora se hace con CSS)
  • dir (ahora se usa ul)
  • hgroup (se usa header o div)

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Las principales diferencias entre HTML4 y HTML5 son las siguientes:

  • Gestiona de mejor forma los errores
  • Soporte para tener aplicaciones web modernas como Netflix en vez de usar Javascript u otros métodos.
  • Semántica mejorada para que se lea mucho más claro
  • Mejoras para los celulares, ya que antes no existían celulares donde se podía ver una página web
  • Soporte de vídeo y audio
  • Entre otras cosas más

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla puede ser considerada, resumidamente, como un grupo de filas donde cada una contiene a un grupo de celdas.

La tabla se crea con componentes como "table", "tr" representa la fila y "td" la celda de la tabla
