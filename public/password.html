Preguntas a contestar

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Es una buena práctica hacerlo para proporcionar una buena experiencia de usuario (información en tiempo real), así el servidor no tiene que hacer doble trabajo y regresar la información para que el usuario lo vuelva a insertar.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Se puede desactivar JavaScript en el servidor y con eso se desabilita las validaciones de seguridad.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Para proporcionar una buena experiencia, y hacer todo más optimo, (el servidor no tiene que regresar y mostrar los errores, todo es en tiempo real). Retroalimentación en tiempo real del usuario.
